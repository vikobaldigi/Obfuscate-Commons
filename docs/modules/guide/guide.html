Terminal Basics (GUIDE)

The command line is less about commands, and more about permission, access, and how machines reveal their structure.

What Is a Terminal?

A terminal is a text-based interface to your computer — a place where you speak directly to the system. It predates the mouse and windows, but it still powers the backbone of servers, routers, research environments, and embedded systems.

It’s not retro. It’s revealing. The terminal lets you see what graphical interfaces conceal: the hierarchy of files, the logic of processes, and the assumptions behind your interaction.

Why the Terminal Still Matters

Graphical interfaces are designed to simplify. That makes them usable — but also limiting. The terminal doesn’t simplify. It exposes. As William Shotts puts it: “The command line makes hard things possible.”

Learning the terminal changes how you understand the system. It gives you vocabulary — not just to use the machine, but to interrogate it.

Literacy and Power

The terminal is a language. You learn verbs like ls, cat, chmod. You read error messages. You trace file paths, redirect streams, inspect process trees.

In doing so, you stop being just a user. You become someone who can intervene — on your own system, or someone else’s. That’s not just skill. That’s power.

Core POSIX Commands

Below are some of the most widely used command-line tools defined by the POSIX standard. These commands are found across Unix, Linux, and macOS systems.

🔍 Inspect and Navigate
  • pwd — Show current directory
  • ls — List files
  • cd — Change directory
  • stat — File information
📁 File Operations
  • cp — Copy files
  • mv — Move/rename files
  • rm — Remove files
  • touch — Create a new file
📄 Viewing and Editing
  • cat — Output file contents
  • head / tail — View top or bottom lines
  • less — Scrollable view
  • nano — Basic text editor
⚙️ Permissions and Identity
  • chmod — Change file permissions
  • chown — Change file ownership
  • whoami — Show current user
  • id — Show user and group IDs
🔧 System Utilities
  • ps — View running processes
  • kill — Terminate a process
  • df — Disk usage
  • uptime — System status
📬 Communication and Pipes
  • echo — Output a string
  • grep — Search within files
  • cut — Extract text by field
  • tee — Output to terminal and file simultaneously
