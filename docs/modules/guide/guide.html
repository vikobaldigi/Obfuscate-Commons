Terminal Basics (GUIDE)

The terminal isn't outdated — it's fundamental. Learn to read the system beneath the surface.

What Is a Terminal?

A terminal is a direct line of communication with your computer. It accepts typed commands, giving you full access to the underlying system.

Unlike a graphical interface, it doesn’t hide complexity — it reveals it. From servers to cybersecurity to data science, the terminal remains central to serious computing work.

Why Use It?

GUIs (Graphical User Interfaces) are easy. But the terminal is powerful. It lets you automate tasks, navigate faster, and understand exactly what your computer is doing.

The more control you want over your digital tools, the more the terminal matters. It’s how professionals manage infrastructure and build systems from the ground up.

What You’ll Learn

This module introduces essential terminal commands, grouped by their purpose. These work across most Unix-like systems (Linux, macOS, etc.).

These commands aren't just syntax. They're a vocabulary for speaking directly to the machine.

Core Commands

🔍 Inspect and Navigate
  • pwd — Show current directory
  • ls — List files and folders
  • cd — Change directory
  • stat — View file metadata
📁 File Management
  • cp — Copy files
  • mv — Move or rename files
  • rm — Delete files
  • touch — Create new empty file
📄 View and Edit
  • cat — Display file contents
  • head / tail — View start or end of file
  • less — Scroll through text
  • nano — Edit files in terminal
🔐 Permissions
  • chmod — Change access rights
  • chown — Change file ownership
👤 Identity
  • whoami — Show current user
  • id — Show user and group IDs
⚙️ System Monitoring
  • ps — List active processes
  • kill — Stop a process
  • df — Disk space usage
  • uptime — System load info
📬 Data Streams
  • echo — Output text
  • grep — Search text
  • cut — Extract text columns
  • tee — Send output to file and screen
