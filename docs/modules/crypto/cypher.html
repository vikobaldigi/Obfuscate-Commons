Substitution Ciphers (CYPHER)

Before encryption was math, it was misdirection. This module explores the roots of secrecy: substitution.

Why Start with Substitution?

The earliest forms of encryption weren’t based on computers or complex algorithms. They relied on swapping one letter or symbol for another to confuse anyone reading the message.

Substitution ciphers are easy to understand and easy to break — but they introduce a key idea: secrecy through transformation.

The Caesar Cipher

Named after Julius Caesar, this cipher shifts each letter in a message by a fixed number. If the shift is 3, A becomes D, B becomes E, and so on.

It only has 25 possible shifts, so it can be broken easily by trying them all. Still, it planted the seed for modern encryption: using a shared secret rule to protect a message.

Why Substitution Breaks

Even if the letters are scrambled, the frequencies of characters don’t change. In English, E shows up more than any other letter.

This leak of information is called frequency analysis — and it turns basic ciphers into puzzles anyone can solve with enough data.

Vigenère and Repeating Keys

The Vigenère cipher uses a keyword to apply different Caesar shifts across the message. The keyword repeats, making each letter shift differently.

This was harder to break — but not impossible. The repeating pattern gives attackers a foothold. It shows how even complex systems can reveal themselves if the pattern is reused.

Mechanical Substitution

In the 20th century, machines like the Enigma cipher added moving parts to the process. These machines could scramble messages in billions of ways.

But when operators reused settings or sent predictable messages, even this was cracked. The lesson? Tools don’t protect us if humans create patterns.

Code vs. Cipher

A code replaces whole words or phrases. A cipher changes letters or bits. Codes need lookup tables. Ciphers follow mathematical rules.

In modern cryptography, ciphers rule. They’re faster, safer, and easier to automate.
